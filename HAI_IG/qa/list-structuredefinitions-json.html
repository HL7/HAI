• HAI Patient This profile is based on the US Core Patient Profile and defines HAI specific patient constraints: A patient identifier representing the facility-assigned patient ID is required. Other identifiers for the person can also be present, such as a United States Social Security Number (id/@root = 2.16.840.1.113883.4.1), a Medicare beneficiary identifier (id/@root = 2.16.840.1.113883.4.338), or secondary patient IDs (id/@root = the appropriate facility OID for patient IDs).   • HAI Custodian Defines NHSN custodian constraints and extensions on the Organization resource for use in HAI reports. NHSN is the custodain of NHSN HAI Reports.
  • HAI Healthcare Facility HAI Healthcare Facility
  • HAI SSI Encounter Key encounter data: The encounter type (inpatient or outpatient) is required. A value for admission date is required. A discharge date may be recorded.
  • Healthcare Associated Infection Report This profile further constraints the C-CDA on FHIR US Realm Header profile. It specifies the generic constraints for all NHSN Healthcare Associated Infection Reports. Further constraints are found in the specialization profiles for single-patient and population summary reports, and in the profiles for specific report types.
  • HAI Single-Person Report Generic Constraints This profile records the constraints for HAI single-person reports. It is used by all numerator reports covered by this guide. It is also used by the Procedure Denominator Report. A single-person report must conform to the Healthcare Associated Infection Report profile. In addition, key data about the patient are recorded in the HAI Patient profile referenced in the subject. The report-specific requirements specify the encounter data to record for each single-person report type. These data always include an identifier for the reporting facility, and usually include the encounter type (inpatient or outpatient), the admission date or encounter date, and the facility unit identifier and type.
  • HAI Surgical Site Infection Report (SSI) This report type records a surgical site infection event. Preferred document title: Surgical Site Infection Report (SSI). Key encounter data: The encounter type (inpatient or outpatient) is required. A value for admission date is required. A discharge date may be recorded. The facility id is required. Other dates and locations: The date of the infection is recorded as effectiveTime in the Infection-Type Observation. Other notes: The procedure id is recorded in the Procedure Details Clinical Statement in an Infection-Type Report. NHSN uses this to establish a link between Procedure and SSI Reports.
  • HAI Procedure Denominator Report Note: The section on Template Ids in this Guide includes a containment table showing all the entries within each report type. This report records a procedure performed. Preferred document title: Denominator for Procedure Report Key encounter data: The encounter type (inpatient or outpatient) is required. If the patient was an inpatient, a value for admission date is required. The facility id is required. Unit identifier and unit type are not reported. Other dates and locations: The date of the procedure is recorded as effectiveTime in the Procedure Details Clinical Statement in a Procedure Report (V2). Other notes: The procedure id is recorded in the Procedure Details Clinical Statement in a Procedure Report (V2). NHSN uses this to establish a link between Procedure and SSI Reports

  • infection-present-at-time-of-surgery This extension records whether or not an infection was present at the time that a surgical procedure was performed.
  • occasion-of-hai-detection This extension records when, in relation to an admission (a surgery stay), an HAI was detected.
  • questionnaire-item-note A note about the questionnaire item.
  • Anesthesia Delivered This profile reports whether anesthesia was administered.
  • MDRO/CDI Observation This observation records whether the primary infection being reported was caused by a multi-drug-resistant organism (MDRO). It is a general or summary observation not associated with any individual pathogen in the Findings Section. If the infection organism was multi-drug resistant, set the value to true. If the infection organism was not multi-drug resistant, set the value to false.
  • Pathogen Identified This profile either represents a pathogen identified or, using a code from the NHSN Pathogens Code value set, records that no pathogens were identified. The value set table provides codes for several unspecified speciesfor example "Acidaminococcus species" (SNOMED CT 131202007, NHSN ACISP)for use when a more precise code is not available. The code for an unspecified species is preferred to a genus-level code. Each Pathogen Identified profile represents a set of information concerning a single pathogen identified. The profile contains two component elements that record the pathogen and its rank in regard to the other pathogens identified, and can contain additional component elements recording drug-susceptibility test findings. For pathogen antibiogram reporting requirements, refer to the NHSN protocol and the Information Data Model (IDM).
  • Revision Associated with Prior Infection Observation This profile records whether or not a surgical procedure was performed as a result of a previous infection. In the NHSN context, this observation only applies to arthroplasty procedures.
  • Spinal Fusion Level This profile records the spinal level of a spinal fusion procedure.
  • ASA Class This profile records the patients physical status using the American Society of Anesthesiologists (ASA) Class Codes. The SNOMED CT representation of ASA Class Codes includes a sixth value (413500003 ASA physical status class 6 for brain-dead patients taken to the operating room to remove organs for transplant). This sixth value is not part of the value set allowed in a submission to NHSN.
  • Diabetes Mellitus This profile records whether the person had diabetes mellitus.
  • Infection Condition This profile records the infection condition being reported.
  • Closure Technique This procedure identifies the closure technique used in surgical procedures.
  • Procedure Details This profile records the detail required in a Procedure Report about a procedure. NHSN uses the procedure id, along with other data, to establish a link between a Procedure Report and an SSI Report; the value of the procedure id must be the same in both the report of a procedure and the report of a surgical-site infection resulting from that procedure. In this profile, the priority code represents the main or primary procedure for the patient in the context of this report.
